# Spring核心概念学习指南

> **学习目标**：从零开始，循序渐进掌握Spring框架的核心概念，为Spring Boot学习打下坚实基础

## 🎯 为什么从核心概念开始？

### 传统学习方式的问题
```
❌ 直接学Spring Boot → 遇到IoC/DI概念 → 不理解 → 学习困难
```

### 我们的学习方式
```
✅ 先理解核心思想 → 再学习具体技术 → 水到渠成 → 轻松掌握
```

**Spring Boot是建立在Spring框架基础之上的，不理解Spring的核心思想，学习Spring Boot就会事倍功半！**

#### 1️⃣ IoC容器与依赖注入核心思想
- **文件**: [[01-IoC容器与依赖注入核心思想.md]]

- **重点**: 传统方式vs Spring方式对比，理解为什么需要IoC和DI
- **实践**: 创建第一个Spring应用，体验自动化依赖管理

#### 2️⃣ Spring注解体系概览
- **文件**: [[02-Spring注解体系概览.md]]
- **目标**: 掌握Spring的"语言" - 注解体系
- **重点**: Bean定义、依赖注入、配置相关注解的分类和使用
- **实践**: 在项目中正确使用各种注解

#### 3️⃣ 快速入门指南
- **文件**: [[03-快速入门指南.md]]
- **目标**: 20分钟创建完整Spring应用
- **重点**: 动手实践，巩固核心概念理解
- **实践**: 从零开始构建REST API

## 🎯 学习方法建议

### 🔄 学→练→会 循环学习法

#### 📖 学一点
- 精读概念解释
- 理解核心思想
- 记录关键要点

#### 💻 练一点
- 按照示例代码实践
- 动手创建Spring应用
- 遇到问题及时调试

#### ✅ 会一点
- 能够解释概念原理
- 能够独立编写代码
- 能够解决常见问题

### 📋 学习检查清单

#### ✅ 基础概念掌握
- [ ] 能够解释什么是IoC（控制反转）
- [ ] 能够解释什么是DI（依赖注入）
- [ ] 理解Spring容器的作用和价值
- [ ] 掌握常用Spring注解的作用

#### ✅ 实践能力培养
- [ ] 能够创建Spring Boot项目
- [ ] 能够使用注解定义Bean
- [ ] 能够实现依赖注入
- [ ] 能够编写简单的REST API

#### ✅ 思维方式转变
- [ ] 从"手动创建对象"转向"让Spring管理对象"
- [ ] 从"硬编码依赖"转向"依赖注入"
- [ ] 从"面向过程编程"转向"面向组件编程"
